# 产品需求文档 (PRD) - Leros

> **最后更新：2026-05-21**
> **版本：3.0**


## 1. 产品定位

Leros 是一个面向企业与团队的 AI 工作协作系统（AI Workspace）。

系统通过以下核心要素构建可持续协作、可执行、可审计的 AI 工作环境：

- **项目（Project）** — 长期工作的核心容器
- **任务（Task）** — 最小执行单元
- **AI队友（Agent）** — 人格化执行者
- **技能（Skill）** — 可复用的能力模块
- **知识库（Knowledge）** — 长期记忆与上下文

Leros 不只是聊天工具。它更像：

- AI Native 的 Jira — 以任务驱动协作
- AI Native 的 Notion — 以知识组织工作
- AI Native 的 Cursor Team — 人机协同开发
- AI Native 的自动化工作系统 — 可持续执行的 AI 流水线

用户可以像组织真实团队一样：创建 AI 队友、分配任务、绑定技能、共享知识、跟踪项目、审计执行过程 — 让 AI 真正参与长期工作流，而不是一次性对话。


## 2. 产品核心概念

### 2.1 核心业务模型

```
任务（Task）  ← AI队友（Agent）执行  ← 绑定技能（Skill）
项目（Project）→ 组织多个任务和 AI队友
知识库（Knowledge）→ 为 AI队友提供上下文
工作台（Workspace）→ 统一工作入口
```

### 2.2 核心实体

| 实体 | 定义 | 职责 |
|------|------|------|
| 任务（Task） | 最小执行单元 | 定义要做什么，谁来做，输出什么 |
| 项目（Project） | 长期工作容器 | 组织任务、AI队友、上下文、记忆 |
| AI队友（Agent） | 人格化执行者 | 理解任务、调用技能、产出结果 |
| 技能（Skill） | 可复用能力模块 | 定义输入、输出、执行流程 |
| 知识库（Knowledge） | 持久化记忆 | 提供领域知识、项目上下文 |

### 2.3 协作关系

```
项目
 ├── 任务 ← AI队友执行 ← 技能
 ├── AI队友（多个，不同角色）
 ├── 知识库（项目专属）
 ├── 项目记忆（历史决策+任务结果）
 └── 长期上下文（目标+规范+业务资料）

平台全局
 ├── 全局技能库
 ├── 全局知识库
 └── AI队友运行时
```

### 2.4 与传统工具的对应

| Leros | 类比 |
|-------|------|
| 项目 | Jira Project + Notion Page |
| 任务 | Jira Issue + GitHub Issue |
| AI队友 | 团队成员（PM/Dev/QA） |
| 技能 | 工具 + API + 工作流 |
| 知识库 | Confluence + 代码文档 |


## 3. 产品信息架构（IA）

### 3.1 一级导航结构

```
工作台
任务
技能
知识库
项目
AI队友

```

### 3.2 导航定义

#### 工作台

用户的统一工作入口。

展示：
- 最近任务
- 待处理事项
- AI 协作动态
- 最近项目
- AI队友状态
- 快捷创建入口

#### 任务

任务是系统中的最小执行单元。

用户可以：
- 创建任务
- 指派 AI队友
- 跟踪执行状态
- 查看执行过程
- 接管任务
- 审核结果

#### 技能

技能是 AI 的可复用能力模块。

例如：PR审查、文档撰写、SQL分析、数据总结、Web搜索、CAD分析。

技能可以被：
- AI队友调用
- 任务直接调用
- 工作流复用

#### 知识库

知识库用于提供长期上下文与企业知识。

支持：文档知识、代码知识、项目知识、FAQ、向量检索、图谱检索。

#### 项目

项目用于组织长期任务、上下文和协作关系。

项目包含：项目目标、长期上下文、任务列表、AI队友、项目知识、项目记忆。

项目是长期工作的核心容器。

#### 个人中心

个人中心是用户管理个人配置和偏好的入口。

包含：
- **个人信息：** 头像、昵称、联系方式
- **账号安全：** 密码修改、登录记录、多因素认证
- **API 密钥：** 管理个人 API Token，用于外部系统集成
- **通知偏好：** 配置消息通知的渠道和频率
- **界面偏好：** 主题、语言、时区等个性化设置
- **操作日志：** 个人操作历史记录
- **使用统计：** 个人任务量、资源使用情况概览


## 4. 工作台

### 4.1 定位

工作台是用户进入系统后的统一工作入口，聚合展示用户关注的核心信息，提供快捷操作入口。

### 4.2 核心模块

**最近任务：** 展示当前用户最近创建或指派的待处理任务。

**待处理事项：** 需要用户审批、验收或接管的任务列表。

**AI协作动态：** 系统内 AI队友的执行动态，包括任务完成、审批请求、异常事件等。

**最近项目：** 用户最近参与的项目列表，支持快速跳转。

**AI队友状态：** 用户常用的 AI队友当前在线状态和忙碌情况。

**快捷创建入口：** 快速创建任务、项目或对话。

### 4.3 个性化

- 支持用户自定义工作台布局
- 支持 Pin 住常用项目和 AI队友
- 支持设置通知偏好


## 5. 任务系统

### 5.1 定位

任务是系统中的最小执行单元。任务驱动 AI队友工作，是人和 AI 协作的基本单位。

### 5.2 任务生命周期

```
创建 → 待分配 → 执行中 → 待审批 → 完成
                    → 失败
                    → 已取消
                    → 已暂停
```

**创建：** 用户创建任务，填写任务描述、目标、预期输出。

**待分配：** 任务等待分配 AI队友或人工执行。

**执行中：** AI队友正在处理任务，或人工正在执行。

**待审批：** 任务执行到需要人工审批的节点，暂停等待审批。

**完成：** 任务成功完成，结果已输出。

**失败：** 任务执行失败，可能由于错误、超时等原因。

**已取消：** 任务被用户取消。

**已暂停：** 任务被用户临时暂停，可恢复。

### 5.3 任务类型

**聊天任务：** 用户与 AI队友的实时对话，不需要持续上下文。

**执行任务：** AI队友执行一个明确的操作（如运行脚本、生成报告）。

**文档任务：** AI队友撰写或编辑文档。

**代码任务：** AI队友编写、审查或修改代码。

**调研任务：** AI队友搜集信息并汇总结论。

**自动化任务：** 由事件或定时器自动触发的任务。

**多Agent协作任务：** 多个 AI队友协同完成的任务。

### 5.4 任务属性

**优先级：** 任务的优先级，影响执行顺序（紧急/高/中/低）。

**截止时间：** 任务需要完成的时间要求。

**标签：** 用于分类和筛选任务的标签。

**关联项目：** 任务所属的项目。

**关联 AI队友：** 执行任务的 AI队友。

**子任务：** 一个任务可以拆分为多个子任务。

**依赖关系：** 任务之间的依赖关系。

**任务上下文：** 任务相关的背景信息、参考资料。

**预期输出：** 任务完成后的产出物描述。

### 5.5 人机协作

**AI执行：** 默认由 AI队友自动执行任务。

**用户接管：** 用户可以随时接管任务，转为人工执行。

**用户审批：** 关键节点需要用户审批后才能继续。

**用户补充上下文：** 用户可以在任务执行过程中补充信息。

**用户中断：** 用户可以中断正在执行的任务，修改指令后继续。

**用户恢复：** 中断或暂停的任务可以恢复执行。

### 5.6 定时任务

支持周期性任务配置（Cron 表达式），用于：
- 每日项目进展摘要
- 定期代码仓库巡检
- 定时知识库同步
- 定时数据分析
- 定时运营报告生成


## 6. 项目系统

### 6.1 定位

项目是系统中组织长期工作的一级核心实体。

项目承载：长期目标、上下文、协作关系、任务、AI队友、知识。

项目是 AI 长期记忆和持续协作的核心容器。

### 6.2 项目组成

```
项目
 ├── 项目信息（名称、描述、目标、状态）
 ├── 任务列表
 ├── AI队友（绑定多个 AI队友，不同角色）
 ├── 项目知识库（项目专属知识）
 ├── 项目记忆（系统自动沉淀）
 ├── 项目技能（项目私有技能）
 └── 项目成员（参与的人类用户）
```

### 6.3 项目能力

**项目上下文：** 维护长期背景信息，包括商业目标、技术架构、项目规范、业务资料。

**项目任务：** 项目下可创建多个任务，任务共享项目上下文。

**项目AI队友：** 项目可以绑定多个 AI队友，例如 PM、架构师、开发、测试、安全审计。

**项目知识库：** 项目可以拥有独立的知识库，存储项目相关的文档、代码、规范。

**项目记忆：** 系统自动沉淀历史决策、任务结果、讨论过程、执行经验。

**项目模板：** 支持将项目保存为模板，快速创建同类项目。

### 6.4 项目生命周期

```
创建 → 进行中 → 暂停 → 已完成 → 已归档
```

- **创建：** 填写项目信息，设定目标和范围
- **进行中：** 项目活跃，任务正在执行
- **暂停：** 项目暂时中止，可恢复
- **已完成：** 项目目标已达成
- **已归档：** 项目关闭，数据保留只读


## 7. AI队友系统

### 7.1 定位

AI队友是可长期协作的 AI 执行角色。AI队友不再是平台中心，而是任务执行角色。

### 7.2 AI队友核心组成

```
AI队友 = 角色 + 技能 + 记忆 + 执行环境
```

### 7.3 角色（Role）

**基础信息：** 名称、头像、描述。

**职责定义：** 负责的工作领域，例如"代码审查"、"问题解答"、"数据分析"。

**工作边界：** 不能做的事情，例如"不允许修改生产环境配置"、"不允许访问敏感数据"。

**系统提示词：** 行为准则和回复风格。

**输出格式：** 输出结果的格式要求。

**拒绝策略：** 超出职责范围或违反边界时的回应方式。

**审批场景：** 哪些操作需要人工审批后才能执行。

### 7.4 技能（Skill）

AI队友绑定的可执行能力模块。

**绑定方式：**
- 从全局技能库选取
- 从项目技能库选取
- 创建自定义技能

**技能权限：** 控制 AI队友对每个技能的使用权限（允许/禁止/需审批）。

### 7.5 记忆（Memory）

AI队友可访问的上下文。

**记忆来源：**
- 项目记忆（当前参与的项目）
- 知识库（已授权的知识库）
- 历史任务（之前执行过的任务）
- 长期偏好（用户设定的偏好）

### 7.6 执行环境（Runtime）

AI队友实际工作的运行环境。

**执行环境配置：**
- 默认模型
- 可用工具
- 工作节点
- 网络策略
- 资源限制

### 7.7 协作模式

**单人模式：** 一个 AI队友独立完成任务。

**主从模式：** 主 AI队友拆解任务，子 AI队友分别执行。

**审查模式：** 审查 AI队友检查执行 AI队友的结果。

**流水线模式：** 多个 AI队友形成执行流水线。

**并行模式：** 多个专业 AI队友从不同角度给出结论。


## 8. 技能系统

### 8.1 定位

技能是平台中的标准能力单元，是平台的核心资产。

技能定义：输入结构、输出结构、所需工具、所需模型、执行流程、权限需求。

### 8.2 技能分类

| 分类 | 说明 | 来源 |
|------|------|------|
| 系统技能 | 平台内置的标准能力 | 平台提供 |
| 项目技能 | 项目私有的能力 | 项目创建 |
| 团队技能 | 组织共享的能力 | 团队创建 |
| 市场技能 | 来自第三方的能力 | 技能市场 |

### 8.3 技能组成

**元信息：** 名称、描述、版本、分类、标签。

**输入Schema：** 技能调用的输入参数定义。

**输出Schema：** 技能输出的结果结构定义。

**执行流程：** 技能的内部执行逻辑（单步工作流或多步工作流）。

**所需工具：** 技能依赖的工具列表。

**所需模型：** 技能推荐或要求的 AI 模型。

**权限声明：** 技能执行所需的权限（文件读写、网络访问、外部API调用等）。

### 8.4 技能类型

**工具技能：** 直接调用外部工具或 API 的技能，如 Web搜索、文件操作。

**推理技能：** 基于 LLM 的推理能力，如代码审查、文本摘要、分类判断。

**工作流技能：** 多步骤编排的技能，如 PR审查工作流、Bug排查工作流。

**MCP服务：** 通过标准协议接入的外部服务能力。

### 8.5 技能管理

- 技能注册：注册新的技能
- 技能分类：按功能领域分类
- 版本管理：管理技能的版本和变更
- 依赖声明：声明技能的依赖关系
- 安全审核：对技能进行安全审核
- 启用和停用：控制技能的可用性
- 使用统计：统计技能的使用频率和效果


## 9. 知识库系统

### 9.1 定位

知识库为 AI队友提供企业内部上下文信息，是 AI 长期记忆的持久化存储。

### 9.2 知识类型

**文档知识：** 技术文档、产品文档、规范文档等。

**代码知识：** 代码仓库结构、API 定义、代码规范等。

**项目知识：** 项目目标、架构决策、业务资料等。

**FAQ：** 常见问题与答案。

**外部知识：** 通过连接器获取的外部知识源。

### 9.3 知识管理

**知识导入：** 从文档、代码仓库、数据库等来源导入知识。

**知识分类：** 按领域、项目、部门等维度分类。

**知识更新：** 支持增量更新和自动同步。

**权限控制：** 控制不同 AI队友和用户可访问的知识范围。

**使用统计：** 统计知识的使用情况。

### 9.4 知识检索

**全文检索：** 基于关键词的全文搜索。

**向量检索：** 基于语义的向量相似度搜索。

**混合检索：** 全文检索与向量检索结合。

**图谱检索：** 基于知识图谱的关系检索。

**RAG 集成：** 与 AI队友执行环境集成的检索增强生成。

### 9.5 知识库级别

| 级别 | 范围 | 说明 |
|------|------|------|
| 全局知识库 | 全平台 | 企业级通用知识 |
| 团队知识库 | 组织单元 | 团队共享知识 |
| 项目知识库 | 单个项目 | 项目专属知识 |
| AI队友知识库 | 单个AI队友 | 个人偏好和记忆 |


## 10. 平台引擎

### 10.1 定位

平台引擎为 AI队友提供真实的工作环境，包括 AI 模型、执行节点、外部连接和基础设施。

### 10.2 AI 模型管理

**模型供应商管理：** 支持多家模型供应商和本地部署模型。

**模型目录：** 维护模型名称、能力标签、上下文长度、成本、延迟、适用场景等信息。

**凭证管理：** 管理 API 密钥、组织 ID 等凭证信息，支持密钥轮换。

**路由策略：** 按 AI队友、任务类型、成本预算选择合适的模型。

**降级策略：** 主模型失败后切换备用模型。

**调用审计：** 记录模型调用、token 用量、耗时、错误和成本。

### 10.3 工作节点

工作节点是 AI队友执行真实任务的基础设施。

**支持的节点类型：**
- 容器沙箱节点：基于容器技术的隔离执行环境
- 远程执行节点：连接到远程服务器执行任务
- Browser 节点：浏览器自动化执行环境（规划中）
- Code Runtime：代码执行运行时（规划中）

**节点能力：**
- 节点健康检查
- 资源容量管理（CPU、内存、磁盘）
- 工作目录管理
- 资源限制
- 执行审计（命令日志、文件变更、执行产物）
- 沙箱生命周期管理

### 10.4 外部连接

外部连接解决"外部系统如何触发任务"的问题。

**支持的外部系统：**
- 代码平台：GitHub、GitLab、Gitee 等
- 事件通知：Webhook、API 回调等
- 业务系统：工单系统、审批系统、告警系统等

**典型通知类型：**
- PR/MR 创建、更新、评论
- Issue 创建、评论
- 监控告警触发/恢复
- 工单状态变更

**连接能力：**
- 事件类型配置
- 请求签名和密钥管理
- 内容映射为任务
- 幂等处理
- 失败重试

### 10.5 IM 身份

IM 身份用于将 AI队友绑定到企业即时通讯工具中的机器人。

**支持的 IM 渠道：** 企业微信、飞书、钉钉、Slack 等。

**典型能力：**
- 私聊消息
- 群聊消息
- @ 消息触发
- 卡片消息
- 文件消息
- 身份映射
- 消息审计


## 11. 企业级治理

### 11.1 多租户隔离

- 租户管理：支持多个企业租户
- 数据隔离：不同租户的数据完全隔离
- 资源隔离：不同租户的资源独立分配

### 11.2 用户和权限管理

- 用户管理：管理平台用户
- 组织管理：管理企业的组织和部门结构
- RBAC 权限：基于角色的权限控制
- 能力权限：控制谁能使用哪些技能和 AI队友

### 11.3 审计能力

- 配置变更审计：记录 AI队友和技能配置的变更历史
- 任务执行审计：记录所有任务的执行过程和结果
- 技能调用审计：记录所有技能的调用情况
- 模型调用审计：记录所有模型的调用情况
- 执行回放：支持回放任务的执行过程

### 11.4 成本管理

- 成本统计：按租户、部门、项目、AI队友、任务统计成本
- 预算限制：设置成本预算告警
- 异常告警：成本异常时告警

### 11.5 数据安全

- 敏感数据脱敏：自动脱敏敏感数据
- 密钥托管：安全地托管 API 密钥等敏感信息
- 数据加密：对敏感数据进行加密存储
- 访问控制：严格控制对敏感数据的访问


## 12. 业务板块规划

### 12.1 规划概述

Leros 面向三大业务板块，提供 AI 驱动的端到端工作协作能力：

| 板块 | 定位 | 阶段 |
|------|------|------|
| 专业文档撰写 | AI 辅助标书/合同/专利等专业文档创作 | **阶段一核心** |
| 软件研发项目 | 需求→开发→测试→发布全流程 AI 协作 | 方向规划 |
| AIOps | 运维监控、告警排查、巡检自动化 | 方向规划 |

三个板块共享同一套平台基础设施（项目、任务、AI队友、技能、知识库、平台引擎），各板块在能力层进行差异化扩展。

**阶段一聚焦「专业文档撰写」**，构建从需求到标准化文档输出的完整闭环，验证 AI 长文档生成、知识库引用、模板匹配、格式输出的核心能力。


### 12.2 板块一：专业文档撰写（阶段一核心）

#### 12.2.1 场景描述

面向企业中的专业文档撰写场景，包括：

- **标书撰写：** 根据招标要求、企业资质、历史案例生成投标文件
- **合同撰写：** 根据业务条款、法务规范、合同模板生成合同草案
- **专利撰写：** 根据技术交底书、专利库检索、格式规范生成专利申请文件
- **其他专业文档：** 可行性报告、技术方案、验收报告等

核心痛点：专业文档格式要求严格、需要引用大量企业知识、需要遵循模板结构、输出格式（Word/PDF）标准化要求高。

#### 12.2.2 核心能力

**知识库引用：**
- 接入企业内部知识库（历史标书、合同库、专利库、法规库）
- 文档级知识检索与段落级引用
- 引用溯源（标注每段内容的来源文档）

**模板系统：**
- 文档结构模板（章节组织、内容占位）
- 格式规范模板（字体、字号、页眉页脚、样式）
- 模板变量（客户名称、项目编号、日期等自动填充）
- 模板版本管理

**网络检索：**
- 行业标准、法律法规检索
- 竞品信息、市场动态检索
- 检索结果自动引用与溯源

**标准化输出：**
- Word 格式导出（保留样式、目录、页眉页脚）
- PDF 格式导出
- 输出格式预览与调整

**人机协作：**
- AI 生成初稿
- 用户逐段审阅修改
- 用户补充上下文和企业特有信息
- 最终确认导出

#### 12.2.3 端到端流程

1. 用户创建文档撰写项目，设置文档类型（标书/合同/专利）
2. 用户提供基本需求描述或上传参考资料
3. AI队友匹配最佳模板
4. AI队友检索知识库获取企业相关知识
5. AI队友检索网络获取行业信息（如需要）
6. AI队友生成文档初稿
7. 用户在编辑器中审阅、修改、补充
8. AI根据用户修改迭代优化
9. 用户确认终稿，导出标准格式（Word/PDF）
10. 文档与项目、任务记录留存，纳入知识库

#### 12.2.4 验收标准

- 支持至少 2 种文档类型模板（标书、合同）
- 知识库引用准确率大于 85%
- 文档初稿生成时间小于 60 秒
- 导出 Word 格式保留完整样式和结构
- 用户完成一篇文档的总耗时降低 50% 以上
- 系统稳定性大于 99%


### 12.3 板块二：软件研发项目（方向规划）

> **状态：规划中，待阶段一完成后推进。**

#### 12.3.1 场景描述

面向软件研发团队，覆盖从需求提出到发布上线的全流程 AI 协作：

- 需求提出与分析
- 任务拆解与分配
- 代码生成与审查
- 集成测试与验收
- 发布管理与上线

#### 12.3.2 核心能力（规划）

- **需求管理：** 需求录入、结构化分析、关联项目上下文
- **任务拆解：** 将需求自动拆解为开发任务，绑定 AI队友
- **代码生成：** 基于任务描述生成代码，关联代码仓库
- **代码审查：** 自动审查代码变更，生成审查意见
- **CI/CD 集成：** 对接 CI 流水线，自动触发构建与测试
- **测试与验收：** 自动生成测试用例，执行测试，生成验收报告
- **发布管理：** 发布清单检查、版本管理、发布记录

#### 12.3.3 端到端流程（规划）

需求提出 → AI 分析需求 → 任务拆解 → AI 编码 → 代码审查 → CI 触发 → 集成测试 → 验收确认 → 发布上线

#### 12.3.4 验收标准（规划）

- 支持完整的软件研发项目闭环
- 代码审查覆盖率达到 80% 以上
- 端到端流程延迟小于 5 分钟


### 12.4 板块三：AIOps（方向规划）

> **状态：规划中，待阶段一完成后推进。**

#### 12.4.1 场景描述

面向运维团队，实现服务器管理、告警处理、巡检排查的 AI 自动化：

- 外部监控系统数据接入
- 服务器批量管理与巡检
- 告警自动分析与根因定位
- 运维知识库与排查规则沉淀

#### 12.4.2 核心能力（规划）

- **外部数据源对接：** 监控系统（Prometheus/Zabbix）、日志平台（ELK/Loki）、APM
- **服务器管理集成：** SSH 执行、配置管理、批量操作
- **巡检规则引擎：** 定时巡检模板、异常检测规则、巡检报告生成
- **告警联动：** 告警接入、自动关联上下文、排查流程编排
- **自动处置：** 常见问题自动恢复、变更回滚、服务重启

#### 12.4.3 端到端流程（规划）

告警触发 → AIOps 接收 → 关联监控/日志/CMDB 数据 → 自动排查分析 → 定位根因 → 生成处置建议 → 人工确认/自动执行 → 事件闭环记录

#### 12.4.4 验收标准（规划）

- 支持至少 2 类外部数据源对接
- 常见告警自动定位准确率大于 70%
- 巡检任务完全自动化


### 12.5 公共平台能力

三个业务板块共享以下平台基础设施：

**项目系统：** 项目创建、上下文管理、模板化

**任务系统：** 任务创建、生命周期管理、人机协作（审批/接管/中断/恢复）

**AI队友系统：** 创建与配置、角色定义、技能绑定、模型配置

**技能系统：** 系统技能注册与绑定、自定义技能

**知识库系统：** 知识导入、分类、检索、权限控制

**平台引擎：** AI 模型调用、工作节点（沙箱/远程）、外部连接（Webhook/API）、IM 身份

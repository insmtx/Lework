# 报价偏差表回填

> 第4步：服务方案完成后回填偏差表和报价表。只回填，不重拆条款，不重命名分项。

## 适用场景

服务方案正文完成后，回填：
1. 底版中的**填空型模块**（封面、投标函、供应商廉洁承诺书、中标服务费承诺书、投标人基本情况表等）
2. 商务条款响应偏差表（含资格证明/商务承诺/报价/实质性条款四类）
3. 技术条款响应偏差表
4. 关键商务和技术指标响应偏差表（含*号条款）
5. 分项报价表

## 核心原则

本步骤回填偏差表和报价表。不重新拆条款、不重命名分项。

**Step 2 应答表** 是完整遍历全文的 200+ 条，**偏差表** 是提炼后的摘要——两者的用途不同：应答表是编制过程的内部参照，偏差表是给评标专家看的最终响应。

### 应答表 → 偏差表的聚合判定

回填前 MUST 先做聚合判定。聚合依据来自招标文件自身的章节结构：

#### 按响应性质分类入表

| Step 2 响应性质 | 入哪个偏差表 |
|---|---|
| 资格证明 | 商务偏差表 |
| 商务承诺 | 商务偏差表 |
| 报价 | 商务偏差表 |
| 技术方案 | 技术偏差表 |
| 实质性条款（有*号标注） | 关键偏差表 |

#### 商务偏差表聚合规则

**以招标文件自身的章节/条款号为聚合单位**。同一章同一节下性质相同的多条要求合为一条，不同性质的分开。

判定方法：
- "招标范围、服务期、服务地点"同在投标人须知 1.3 下 → 合并为一条
- "付款方式"与"知识产权"分属合同不同条款，性质不同 → 各自成条
- "廉洁承诺"与"保密协议"分属不同附件，独立主体 → 各自成条
- 同一资格要求的不同条款（如资格要求1、资格要求2）→ 各自成条

#### 技术偏差表展开规则

**以招标文件技术章节的子条款编号为最小单位**。编号到哪一级就展开到哪一级。

判定方法：
- 招标文件编号为"3.2.1.2"的条款 → 独立成条
- 编号为"3.2.1.3"的条款 → 独立成条
- 不允许跨子条款编号合并（如把 3.2.1.2 和 3.2.1.3 压成一条）
- 若多个连续条款描述同一功能域（如 3.2.2.2.6-3.2.2.2.8 均为门户的辅助功能），可聚合为一条，条款号写作"第五章/3.2.2.2.6-3.2.2.2.8"

#### 关键偏差表*号条款判定

**以招标文件实际标注为准**，不以评标办法中关于*号的描述为准。

判定流程：
1. 遍历第五章"服务标准及要求"全文，搜索文本中实际出现的星号 `*`
2. 有实际*号标注的条款 → 逐条填入关键偏差表
3. 招标文件全文无实际*号标注 → 填入一行"招标文件无标*号项"，响应情况填"/"，偏差说明填"无偏差"
4. **MUST NOT** 把"关于*号的规定"（如"标注星号（*）的条款不允许有负偏离"）当作*号条款填入

### 用户参考案例

若用户提供了参考标书（如脱敏标书），应在聚合判定前先读取，提取其偏差表的归类风格和粒度作为参照，再结合招标文件实际结构决定聚合方式。

若用户未提供参考案例且聚合粒度存在歧义，应主动询问用户偏好。

### 回填范围

前置阶段已经确定：
- 分项报价表中的"分项名称"（Step 1 速查表固定）

本步骤完成：
- 偏差表的条款聚合 + "投标文件响应情况" + "偏差说明"
- 分项报价表的"服务内容" + "备注" + "单价" + "总价"
- 填空型模块的内容替换

## 禁止事项

- 不得新增/删除/重排偏差表条款号
- 不得改动人工已确认的商务或技术归类
- 不得重新提取或修改分项名称
- 不得编造单价/总价
- 不得编造服务方案中没有的能力
- 不得把评分办法/模板/流程说明重新拆进偏差表

## 输入材料

必须输入：
- 已确认的商务条款响应偏差表条款号清单
- 已确认的技术条款响应偏差表条款号清单
- 已确认的分项名称清单
- 已完成的服务方案正文
- 已完成或待填写的分项报价表模板

用户提供：单价、总价、报价策略、是否含税、是否含授权/实施/培训/售后/差旅/质保

如果用户没有提供价格，不得自行编造，应输出"待用户提供"。

## 偏差表回填规则

### 偏差表模板行处理

底版中偏差表的空模板通常只有表头行 + 一个空白占位行。回填方式：
- 保留表头行不变
- 删除空白占位行
- 从第二行开始逐条追加聚合后的响应行

偏差表中的"招标文件章节及条款号"来自上述聚合判定结果，**MUST NOT** 无依据地新增或删除。

### 投标文件响应情况写法

统一格式："完全响应，无偏差。见【投标文件对应章节】。"

**引用精度要求**：引用位置 MUST 精确到子节编号（如"七、服务方案（二）1 理解与响应"），不能只写到章级（如"见七、服务方案"）。应引用 Step 4 方案中实际存在的子节标题，方便评标专家直接定位。

示例：
- 完全响应，无偏差。见一、投标函。
- 完全响应，无偏差。见五、资格证明文件。
- 完全响应，无偏差。见七、服务方案（二）1 理解与响应。
- 完全响应，无偏差。见八、服务保障及措施（二）部署实施服务。

### 偏差说明写法

偏差说明必须解释"为什么无偏离"，不能只写"无偏差"。保持简洁，1-2 句话即可。采用"承诺/已提供 + 具体内容 + 故无偏离"的句式。

商务偏差说明示例：
- 已提供营业执照等全部资格证明材料，满足投标人资格要求，故无偏离。
- 投标报价未超过最高限价且逐项填写无缺漏，故无偏离。
- 承诺缴纳投标保证金并按要求提供凭证，故无偏离。
- 承诺接受合同全部条款（含付款、验收、违约、保密及知识产权），故无偏离。

技术偏差说明示例：
- 方案已逐项说明该平台的功能模块与技术实现方式，满足招标技术指标，故无偏离。
- 方案支持招标要求的部署环境与系统适配，故无偏离。
- 已制定对应的实施、测试、验收和培训安排，满足服务标准及交付要求，故无偏离。

| 服务范围 | 投标函、服务方案 |
| 服务期或工期 | 投标函、项目进度计划 |
| 服务地点 | 投标函、实施方案 |
| 合同付款 | 服务保障、综合承诺函 |
| 验收和交付 | 服务保障、验收与交付物章节 |

## 填空型模块回填规则

底版中有些模块不是表格，而是段落中的占位符需要替换。这些模块的信息来源是 Step 1 速查表和用户提供的投标人信息。

| 模块 | 回填内容 | 信息来源 |
|------|---------|---------|
| 封面 | 项目名称、投标人名称、日期 | Step 1 速查表 + 用户提供 |
| 投标函 | 招标人名称、项目名称、投标报价金额 | Step 1 速查表 + Step 5 报价 |
| 供应商廉洁承诺书 | 投标人签章（模板文案保持原样） | 底版原模板已有完整文案 |
| 中标服务费承诺书 | 投标人签章（模板文案保持原样） | 底版原模板已有完整文案 |
| 投标人基本情况表 | 公司名称、注册资本、地址等 | 用户提供的公司资质材料 |

回填方式：精确替换段落中的占位符文本（如"（项目名称）"→实际项目名称）。**MUST NOT** 修改模板的格式、表格结构或固定文案。

## 分项报价表回填规则

### 分项名称
前置阶段已确定，本步骤只校验，不改名。

### 服务内容
从服务方案对应分项中压缩生成。

推荐写法："产品定位 + 核心能力 + 服务范围 + 部署/授权/培训/售后说明"。

示例结构："面向【业务对象/使用场景】，提供【核心功能1、核心功能2、核心功能3】等能力，支持【部署方式/系统适配/权限/集成】；服务内容包含【安装部署、配置调试、培训、验收及质保期服务】。"

### 备注
备注用于说明：是否含授权、部署实施、培训、售后、质保、差旅、永久授权或订阅授权等。

示例：
- 满足招标文件技术标准/服务要求相关条款，含部署实施、培训及质保期服务。
- 含软件永久授权、安装部署、系统调试及验收支持。
- 含定制开发、现场实施、用户培训及质保期内售后服务。

### 单价和总价
规则：
1. 单价必须由用户提供或来自已确认报价策略
2. 总价根据单价和数量计算
3. 最终合计必须与投标函总价一致
4. 如果不一致，必须提示用户调整
5. 不得自行编造报价

## 输出格式

### 商务条款响应偏差表

| 序号 | 招标文件章节及条款号 | 投标文件响应情况 | 偏差说明 |
|---|---|---|---|

### 技术条款响应偏差表

| 序号 | 招标文件章节及条款号 | 投标文件响应情况 | 偏差说明 |
|---|---|---|---|

### 分项报价表

| 序号 | 分项名称 | 服务内容 | 单价 | 总价 | 备注 |
|---|---|---|---|---|---|

## 质量检查

- 是否没有改动条款号
- 是否没有新增或删除条款
- 是否没有改动分项名称
- 偏差表引用位置是否真实存在于服务方案或投标文件框架中
- 偏差说明是否解释了为什么无偏离
- 报价表服务内容是否来自服务方案
- 备注是否说明报价覆盖范围
- 单价和总价是否来自用户或明确报价策略
- 报价合计是否与投标函一致

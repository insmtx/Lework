# 服务方案写作

> 第3步：在投标文件框架和商务技术条款清单已确认后，按模块向用户索要材料并编写服务方案正文。

## 适用场景

框架、分项名称、商务技术条款清单已确认，开始写正文。

**输出目标**：底版 docx 中的自拟模块（通常为"八、服务方案""九、服务保障及措施""十、售后服务和质保期服务计划"）。**MUST NOT** 将方案写成独立的 .md 或 .docx 文件——必须直接写入底版对应章节。其他有固定模板的模块（投标函、承诺书、资格证书等）不在本步骤写入范围。

## 核心目标

服务方案必须同时服务于三件事：

1. 满足评分项
2. 支撑技术条款响应偏差表
3. 支撑分项报价表的服务内容

部分商务条款如服务范围、服务期、验收、保密、安全、知识产权，也可由服务方案或服务保障章节支撑。

## 输入材料

必须输入：
- 完整招标文件
- 第一步输出的投标文件框架和分项名称清单
- 第二步输出的商务标、技术标拆分表

可选输入：招标澄清文件、产品资料、公司简介、案例、架构图、产品截图、人员信息、实施计划、售后政策

## 写作前材料收集原则

按服务方案模块分别要材料，不一次性向用户索要所有材料。

### 项目理解模块需要用户补充
如招标文件信息不足，可索要：客户背景、项目背景、客户当前痛点、客户已有系统、项目建设目标、业务使用对象、典型业务场景、沟通纪要

### 总体架构模块需要用户补充
可索要：拟投产品总体架构、系统模块关系、数据流/业务流/权限流、部署架构、技术栈、与第三方集成方式、国产化适配要求（如招标文件要求信创/国产化）

### 分项方案模块需要用户补充
针对每个分项名称单独索要：产品定位、核心功能、技术参数、是否已有成熟产品、是否需要定制开发、授权方式、部署方式、交付边界、与招标技术要求的对应关系

### 数据和 AI 能力模块需要用户补充
可索要：数据源类型、文件/数据库类型、知识库方案、RAG 方案、模型能力、Agent 能力、OCR/图像/语音等工具能力、模型路由/Prompt/评测/日志

### 实施部署模块需要用户补充
可索要：服务期/工期、实施阶段划分、里程碑、驻场安排、部署环境、服务器/OS/DB/中间件、测试计划、验收流程、交付物清单

### 项目团队模块需要用户补充
可索要：项目经理、技术负责人、产品负责人、开发/测试/实施/运维人员、人员证书、类似项目经验

### 培训和售后模块需要用户补充
可索要：培训对象、内容、方式、次数、考核方式、售后服务期、响应时间、故障等级、运维方式、服务热线/工单机制

### 知识产权和成果模块需要用户补充
如招标文件要求论文/专利/源码/文档移交，可索要：知识产权承诺口径、源码是否移交、文档清单、论文/专利/软著产出计划、甲方署名或权属要求

## 服务方案通用结构

根据招标文件实际要求调整，建议保持以下结构：

1. 项目总体理解
2. 建设目标与需求分析
3. 总体服务方案或总体技术方案
4. 系统总体架构或服务总体框架
5. 业务流程或实施流程
6. 重点难点分析及应对措施
7. 分项一方案
8. 分项二方案
9. 分项三方案
10. 更多分项方案
11. 部署实施方案
12. 项目组织与进度计划
13. 质量保障措施
14. 安全保障措施
15. 风险控制措施
16. 验收与交付物
17. 培训与知识转移
18. 售后服务和质保期服务计划
19. 关键技术能力或研发能力
20. 其他评分项专项响应

## 每个分项方案的写法

每个分项必须按同一套结构写，便于后续补技术偏差表和报价表。

推荐结构：
1. 分项理解与响应
2. 建设目标
3. 功能方案
4. 技术实现
5. 部署与交付
6. 验收方式

写作公式："招标要求 / 客户问题 → 我方理解 → 建设目标 → 功能能力 → 技术实现 → 交付效果"

## 写作要求

1. 章节标题要便于偏差表引用
2. 尽量与技术条款号、分项名称、评分项形成对应关系
3. 不要只写功能清单，要写业务价值和实现方式
4. 不要编造用户未提供的产品能力、人员、案例、证书、报价
5. 对用户未提供的信息，用"待补充"或向用户提问
6. 服务方案应能直接支撑后续"完全响应，无偏差"的偏差表写法
7. 如果招标文件要求图表，可在对应章节标记"建议插入架构图/流程图/计划表"

## 输出方式

可以分阶段输出：
1. 先输出服务方案详细目录
2. 再按章节逐段写正文
3. 每写一个分项前，列出该分项需要用户补充的信息
4. 用户补充后再生成正文
5. 生成正文后标记可支撑的技术条款和报价分项

## 服务方案完成后的检查清单

- 是否覆盖所有分项名称
- 是否覆盖技术标拆分表中的技术条款
- 是否覆盖评分项
- 是否包含实施、验收、培训、售后
- 是否有对应章节可供偏差表引用
- 是否能提炼出分项报价表服务内容
- 是否没有和商务承诺、报价范围冲突

# 招标文件速查表

> 第1步：通读招标文件全文，把散落在各处的关键约束和决策信息汇总成一张速查表。这张表是后续所有步骤的参考基准。

## 为什么第一步做这个

拿到招标文件后，投标人最需要的是一个"快速参考面板"——不用反复翻几百页文档去找项目名称、截止时间、保证金金额、废标红线。这些信息分布在招标公告、投标人须知前附表、评标办法、合同条款等各处，一次性整理出来，后续写标书时直接引用。

## 操作流程

### 第一步：通读招标文件全文

用 pandoc 或 python-docx 提取招标文件全部文本内容，重点读取：
- 招标公告 / 投标邀请书
- 投标人须知（特别是前附表）
- 评标办法（评分标准和权重）
- 合同条款（关键商务条件）
- 技术标准 / 服务要求
- 投标文件格式章节

### 第三步：输出待确认事项

速查表填写完毕后，标注其中需要用户确认或决策的项：
- 我方是否满足所有硬性门槛？
- 是否需要补充业绩、人员、证书？
- 报价策略（是否含税、税率、是否含实施/培训/差旅）
- 哪些条款需要销售/商务进一步沟通？

### 第四步：用户确认

⛔ 阻塞。速查表确认后，进入下一步（提取标书底版）。

## 禁止事项

- 不得跳过任何章节
- 不得编造信息填补空白
- 不得在尚未确认时就开始填写标书内容

## 输出

一份包含 A~F 六部分的招标文件速查表，标注来源章节和条款号。

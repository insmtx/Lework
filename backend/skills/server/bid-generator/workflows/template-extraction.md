# 标书底版提取

> 速查表确认之后执行：从招标文件 docx 中提取"投标文件格式"部分，作为标书底版。

## 前置条件

- 第1步速查表已完成且用户确认
- 招标文件中有明确的"投标文件格式"或"投标文件组成"章节

## 核心原则

**不要仿造，要搬用。** 招标文件的投标文件格式章节包含封面、目录框架、投标函、报价表、偏差表、承诺书等全部内容——格式、表格、固定文案 100% 保留原样。

## 操作流程

### 第一步：解压招标文件，定位 body 元素索引

用 python 解压 docx（本质是 ZIP），解析 `word/document.xml`，列出 `<w:body>` 下所有直系子元素，按索引展示其类型和首段文本：

```python
from docx import Document  # 或用 xml.etree 直接解析
# 遍历 body 子元素，列出 index / type / 首行文本
```

### 第二步：找到投标文件格式章节的开头

在 body 子元素列表中，找到对应"投标文件格式"章节标题的段落索引。

常见的定位方式：
- 搜索段落文本为"投标文件格式"（也可能为"投标文件组成"、"响应文件格式"）
- 结合样式判断（通常为 Heading 级别）
- 确认该段落之后紧跟的是封面、投标函等标书内容

**以实际招标文件内容为准，不预设关键词。**

### 第三步：确定提取范围

- **起始索引**：章节标题的**下一个**元素（跳过标题行本身）
- **终止索引**：通常到文档末尾即可（body 最后一个元素是 `sectPr` 页面设置，会自动保留）
- 如果招标文件"投标文件格式"之后还有其他不相关内容（如附录），则需指定终止索引

### 第四步：调用提取脚本

```bash
python3 ${SKILL_DIR}/scripts/extract_docx_range.py \
  <招标文件.docx> <输出底版.docx> \
  --from <起始索引>
```

`--from` 之后的索引是 body 子元素的 0-based 位置（即第二步找到的标题索引 + 1）。

### 第五步：验证底版内容

提取成功后，**MUST** 验证底版第一页是否正确。列出底版前 5 个段落的文本和样式，确认：
- 第一页是封面模板（含"投标文件"字样、项目名称占位符、投标人名称占位符、日期占位符），而非"投标文件格式"章节标题
- 若第一页为空白页或错误内容，重新确认起始索引并重新切出

### 第六步：分析底版内容

用 python-docx 解析底版，列出完整结构。标注：
- 哪些模块有固定模板（接填写）
- 哪些模块仅标题占位（需自拟）
- 分项名称清单（从分项报价表原文读取）

### 第七步：确认

将底版 docx 和分项名称清单展示给用户确认。确认后进入第2步。

## 禁止事项

- 不得用 python-docx 手写代码仿造模板
- 不得自行设计或修改模块顺序
- 不得重命名分项名称
- 不得在底版中写入任何投标人信息

## 输出

1. **标书底版 docx** — 从招标文件直接切出的投标文件格式部分
2. **分项名称清单** — 从分项报价表中读取，原文不变
3. **自拟模块清单** — 标记哪些模块需要后续编写
